# HTML Slide Renderer Skill

## Purpose
Render carousel slides as 1080x1080 PNG images using HTML+CSS+Playwright.
Text is 100% exact from Claude output. Colors from project media_config. Cost: $0.
Generic — works for any project or niche. All values come from media_config.

## When to use
- Any carousel slide with copy generated by Claude
- Projects with image_style = "typographic" in media_config
- Default provider for ALL new projects unless client requests artistic images
- When text must be pixel-perfect — no interpretation, no errors, no wrong language

## When NOT to use
- Client uploads their own images — no rendering needed
- Project visual_style = "photorealistic" or "illustration" — use Ideogram or Nano Banana
- Slide needs a real photo background — use external provider

## Stack
- playwright (chromium) for HTML-to-PNG rendering
- Space Grotesk via Google Fonts (@import url in every HTML template)
- S3Service.upload_bytes() for storage
- PlaceholderProvider as fallback if playwright fails — never crash the request

## Installation (add to both Dockerfile.dev and Dockerfile.prod)
RUN pip install playwright --break-system-packages
RUN playwright install chromium --with-deps

## Critical rules
1. Always <meta charset="UTF-8"> in every HTML slide
2. Viewport must be exactly 1080x1080 before screenshot
3. Use wait_until="networkidle" so Google Fonts loads before screenshot
4. ALWAYS escape HTML special chars before injecting text into HTML
   Use Python's html.escape() on every headline and subtext field
   & → &amp; | < → &lt; | > → &gt; | " → &quot;
5. Never hardcode colors — always read from media_config
6. Always catch exceptions and fall back to placeholder silently
7. Each layout uses position: relative on container and position: absolute on footer

## media_config fields used

| Field | Default | Description |
|---|---|---|
| image_bg_color | #0a0a0a | Background hex color |
| image_primary_color | #ffffff | Accent color (dividers, labels, CTA border) |
| image_secondary_color | #ffffff | Body text color |
| brand_handle | project.slug | Social handle shown in footer (without @) |
| image_cta_url | project domain | URL displayed on CTA slide |
| image_cta_text | optional | Custom CTA text. Default: "FREE → {image_cta_url}" |

All values are project-specific and come from the DB. No hardcoded per-project defaults.

## Three layouts — chosen automatically by slide number

### Slide 1 — Hook layout (maximum visual impact)
- accent-bar: 70px wide, 6px tall, primary color, 50px margin-bottom
- headline: 76px, font-weight 800, secondary color, max-width 900px
- subtext: 36px, font-weight 400, rgba(255,255,255,0.65), max-width 820px
- footer (absolute bottom 55px): @{brand_handle} left in primary color | "N / total" right

### Slides 2 to N-1 — Content layout (structured, numbered)
- slide-label: "0{N}" 18px, font-weight 700, primary color, letter-spacing 4px
- headline: 66px, font-weight 800, secondary color, max-width 900px
- divider: 80px wide, 5px tall, primary color, 40px top/bottom margins
- subtext: 34px, font-weight 400, rgba(255,255,255,0.7), max-width 880px
- footer (absolute bottom 55px): @{brand_handle} | counter

### Last slide — CTA layout (centered, call to action)
- Centered layout (align-items center, text-align center)
- headline: 70px, font-weight 800
- subtext: 32px, rgba(255,255,255,0.6), margin-bottom 70px
- cta-box: border 3px solid primary color, uppercase, 26px, letter-spacing 3px
  Text: media_config.image_cta_text OR "FREE → {image_cta_url}"
- brand-handle: absolute bottom 55px, rgba(255,255,255,0.35)

## Typography
- Font family: Space Grotesk (Google Fonts) — import in every template
- Font smoothing: -webkit-font-smoothing: antialiased on body
- word-wrap: break-word on all text elements
- Container padding: 90px all sides

## Setting up a new project
In the project media_config:
{
  "image_provider": "html",
  "image_bg_color": "{project background hex from brand onboarding}",
  "image_primary_color": "{project accent color hex}",
  "image_secondary_color": "#ffffff",
  "brand_handle": "{instagram handle without @}",
  "image_cta_url": "{project domain}",
  "image_cta_text": "{optional — leave empty for default}"
}
All values come from Tab 4 'Marca y Visual' in ProjectFormDialog.
